Appendix C: Per-Table Validation Checklist

Each table must pass all 8 validation checks before the batch is signed off:

  • Schema match — Column names, data types, and nullability match source definition
  • Row count — Exact row count match between source and target
  • Checksum — Hash comparison on key columns matches between source and target
  • Null patterns — Null distribution per column matches source patterns
  • Min/Max values — Boundary values for numeric and date columns match source
  • Sample records — 0.1% random sample record-level comparison passes
  • Constraints — Primary keys, foreign keys, and check constraints verified
  • Indexes — Index definitions and statistics confirmed (where applicable)
